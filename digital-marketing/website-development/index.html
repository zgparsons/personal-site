Relational Databases

What is a Database?

A database is a set of data which is stored by a computer. This data is usually stored in an organised or logical way, often with the aid of tables that are stored electronically on the computer or a server. Oftentimes there will be many different variables which affect how the data might be read or understood by a user (more about this later).

In the above database we see some typical elements of a flat-file database. It is a series of columns and rows, in one table, organized in a tabular format. The columns are labelled: Company (a company name), Contact (the name of a contact within that company), and Country (the country in which the company, and contact, are based). As mentioned though databases can quickly become a lot more complex than this which is where relational databases come in.

What is a Relational Database?

When a database begins to use a more complex structure to allow users to identify and access data in relation to other pieces of data in the same database it becomes a relational database. To follow on from the example above, if the person using the database started to meet more than one contact from each company, and they also wanted to start including each contact's job title and/or role it might be helpful for them to begin to seperate out certain pieces of data in order for the information to be more helpful to them.

If this person wanted to contact only the marketing departments of each company in their database, but knew that some of their contacts were from the operations (or HR, or any other departments), then being able to seperate out and access only the contacts from marketing departments would be a lot more helpful.
