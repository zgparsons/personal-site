Relational Databases

What is a Database?

A database is a set of data which is stored by a computer. This data is usually stored in an organised or logical way, often with the aid of tables that are stored electronically on the computer or a server. Oftentimes there will be many different variables which affect how the data might be read or understood by a user (more about this later).

In the above database we see some typical elements of a flat-file database. It is a series of columns and rows, in one table, organized in a tabular format. The columns are labelled: Company (a company name), Contact (the name of a contact within that company), and Country (the country in which the company, and contact, are based). As mentioned though databases can quickly become a lot more complex than this which is where relational databases come in.

What is a Relational Database?

When a database begins to use a more complex structure to allow users to identify and access data in relation to other pieces of data in the same database it becomes a relational database. To follow on from the example above, if the person using the database started to meet more than one contact from each company, and they also wanted to start including each contact's job title and/or role it might be helpful for them to begin to seperate out certain pieces of data in order for the information to be more helpful to them.

If this person wanted to contact only the marketing departments of each company in their database, but knew that some of their contacts were from the operations (or HR, or any other departments), then being able to seperate out and access only the contacts from marketing departments would be a lot more helpful.

Logic Gates

At their most basic level logic gates are used in computer systems to make decisions, usually based on the input from a user. Although people use computers all of the time, most are unlikely to be aware of this process going on because they will be using software designed for their purpose.

As an aside, this is an example of abstraction - an idea which features heavily in the computer science world (and IRL too). Put simply, abstraction is representing the essential features of something useful without including any explanations or background details about how it works. Abstraction reduces complexity for the end user (and also for learners and software engineers even).

A transistor is an electronic circuit which acts like an on / off switch. Electricity will either flow through the circuit or not, depending on the input from a user, thereby creating an on or an off state. A computer uses a similar idea, but with binary signals and usually with many different instances occuring at the same time. Broadly we call these Logic Gates, examples below.

Or

An 'Or' logic gate will give a + if one or both conditions are met.

In a lighting circuit, with two switches side by side, the light would be on if either or both switches are pressed.

And

An 'And' logic gate will give a + if both conditions are met.

In a lighting circuit, with two switches side by side, the light would be on if both switches are pressed.

Not

A 'Not' logic gate will give a - if condition(s) are met.

In a lighting circuit, with one switch, the light would be off if the switch is pressed.

Xor

Similar to the or logic gate, the 'Xor' logic gate will give a + if one condition is met. It differs because if both are met it will give a -.

In a lighting circuit, with two switches side by side, the light would be on if either switch was pressed to an active position, but it would be off if both switches were in the active position.
