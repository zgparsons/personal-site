What is a Content Management System?

A content management system is a software application that we use to publish digital content. Content can be added, updated and deleted via the CMS (and typically an easy to use User Interface, or UI) rather than via files. Website content will typically include text primarily, and then also embedded graphics, images & photos, videos, audio files, maps, and other applications (which use different types of program code), which will all be displayed to the user. A successful CMS must therefore be able to store and deliver all of these file types.

According to user statistics (and Wikipedia) Wordpress is the most popularly used Content Management System. It is used by over 28% of all websites, and 59% of sites known to be using a CMS. Joomla and Drupal are two other market leaders. There are many others available too. More recently, and increasingly, some CMS systems also contain 'Drag and Drop' interfaces. An example of this is Weebly. They can be easier to use for people with less experience in website creation, however they can also be a bit limited in the possibilities available.

Weebly is also a good example of a free (or almost free) CMS. If you are happy for Weebly to host your website (which might include them advertising), and if you do not mind having a weebly domain name, rather than a custom one, it is a free service. (More about both website hosting and domain names later on...)

Common Features of CMS
  • Design is seperate from content
  • SEO-Friendly URL structuring
  • Shared resources
  • Regular updates - features & security
  • Mobile ready
  • Archive capabilities
  • Database driven
  • Remote access
  • Easy to learn/use - many tutorials & walkthroughts available online
  • Cost-effective (Wordpress is open source)
  • Easy to link to CRM (or ecommerce, or other) systems

CMS Advantages & Disadvantages Compared to Static Websites
  • No need to learn HTML, CSS, Javascript etc.
  • Usually there is version control built into CMS
  • CMS allows for different levels of editing permissions
  • Can be easier to create a unified look & feel, via using themes & plugins
  • Added functionality is usually a lot easier to integrate
  • CMS will have a WYSIWYG (What You See is What You Get) editor
  • +
  • CMS may have limited ability to create functionality not originally envisioned by the CMS creator
  • A CMS may increase the need for special expertise or specific training for content creators/authors

What is Web Hosting?

The files which make up a website need to be stored somewhere, and where they are stored needs to be accessible to all via the World Wide Web.

Web hosting companies provide a range of different solutions for this. Essentially each solution offers some physical computing space which has internet connectivity (often this means space on a remote server) where the files of a website are uploaded to and stored.

These hosting companies are not the only option though. It is possible to host your own website files. There are some advantages and disadvantages to both of these options:

Advantages of Self Hosting

  • Control
  • Ability to back-up and access files more freely
  • Manage your own database
  • Manage your own hardware

Disadvantages of Self Hosting

  • Site might not be scalable
  • Spikes of traffic might cause site to go down
  • Personal ability is being relied on
  • May be expensive equipment and internet costs

Free Web Hosting?

Yes, there is also versions of free hosting available which sounds great, but does have some key disadvantages. They are in fact free, but for example it will most likely have advertising, you will be unable to use a custom domain name, and there will be no direct support, and it is unlikely there will be very responsive forum support. You may also see limitations to the bandwidth and speed available to your files.

What is a Domain Name?

Domain Name System, DNS, is a friendly naming system for giving addresses to web servers and web pages. To access a website, you need to give your browser some information - the main part of this information is the location where the website files are stored. This information is given as an IP address, which looks like this: 172.16.254.1 in IPv4, and 2001:db8:0:1234:0:567:8:1 in IPv6 (copy/pasted from Wiki). Essentially the Domain hides the IP address, and makes it easier for hoomans to recognise/understand.

  1. Top level domain = .co.uk / .com / .gov.uk / etc.
  2. Mid level = eg. Facebook / surreycc / zacharyparsons
  3. Machine name = www.

Why Have a Website?

Certainly in terms of Digital Marketing, the main audience that you are expecting is the customer.

With this in mind, the company's website can spill into each 'P' in the marketing mix (laid out below, but more about that in KM2).

  • Place - Product should be available where your target audience can find it easiest.
  • Promotion - Advertising and PR.
  • Price - Should represent good value for money.
  • Product - Should fit the task which consumers want it for.
  • Processes - The delivery of your product or service is done with the customer present, how the service is delivered is part of what the consumer is paying for.
  • People - Companies are reliant on the people who run them - front line to MD.
  • Physical Evidence - Almost all services include some physical elements even if the bulk of the product is intangible.

The purposes of a website are:

To be found | To build relationships | To be an ecommerce platform | To offer a service or information | To offer entertainment | Blogs & updates

The Importance of User Experience (UX) & User Interface (UI)

The User Interface, or the UI, governs the User Experience, or the UX. The website should be created with the user in mind primarily. Important things include having a simple navigation system, analysing bounce rates, and ensuring the site meets accessibility standards.

Having a simple navigation will make the website easy to understand for new visitors (who might also be new customers), it will help existing customers feel comfortable (and hopefully help to retain them), and it will hopefully limit the number of bounces...

The bounce rate is the percentage of visitors to a particular website who navigate away after only a few seconds.

Examples of good and bad bounce rates:

  • 26 - 40% | V. good
  • 41 - 55% | Still good
  • 56 - 70% | Hmmm...
  • 70% + | Need to make some changes to UI

What is Web Accessibility?

Accessibility on the web is about removing barriers that prevent interaction with, or access to, websites by people with disabilities.

For example, for some people to view websites might be more difficult, but it can be a key source (sometimes the only source) of information available. For people with visual impairments, motor/mobility difficulties, inability to use hands, deafness or hearing impairments, types of seizures, or developmental/learning/cognitive disabilities, it is important to make the web as accessible as possible.

This website gives a full report on how accessible a website is, and any issues which it might have.

The Equality Act 2010 states that a website owner must make sure that their website can be accessed by people with disabilities.
